# NSK OpenData Bot

CLI-агент для работы с открытыми данными и системами жизнеобеспечения Новосибирска.
Разработан для ЦИИ НГУ как демонстрация агентного подхода: «семантика + кэш + реальное время».

---

## Быстрый старт

```bash
# Установка
pip install -e .

# Первичная загрузка данных
bot update --all        # открытые данные (CSV, ~1–2 мин)
bot power update        # отключения ЖКХ (мгновенно)

# Задать вопрос
bot ask "сколько парковок в Центральном районе"
bot ask "есть ли отключения электричества сегодня"
```

## Команды

### `bot ask "<запрос>"`
Задать вопрос на русском языке. Бот автоматически определяет тему и тип запроса.

```bash
# Подсчёт
bot ask "сколько школ в Новосибирске"
bot ask "сколько аптек в Советском районе"

# Группировка
bot ask "сколько парковок по районам"
bot ask "школы по районам"

# Список с фильтром
bot ask "покажи библиотеки в Ленинском районе"
bot ask "остановки в Советском районе"
bot ask "аптеки на улице Ленина"

# Топ
bot ask "топ-10 парковок по числу мест"

# Отключения электроснабжения — в реальном времени
bot ask "отключения электричества сегодня"
bot ask "есть ли свет в Дзержинском районе"
bot ask "плановые отключения света на этой неделе"
bot ask "история отключений электроэнергии за неделю"
```

### `bot power` — отключения систем жизнеобеспечения

Данные в реальном времени с официального портала мэрии 051.novo-sibirsk.ru.
TTL = 30 минут. История хранится 7 дней.

```bash
bot power update                         # загрузить свежие данные
bot power status                         # электроснабжение: сейчас
bot power status --all-utilities         # все системы (вода, газ, тепло, электро)
bot power status --district "Советский район"
bot power planned                        # запланированные на сегодня
bot power planned --district "Центральный район"
bot power history                        # история по дням за 7 дней
bot power history --days 3               # история за 3 дня
```

**Что показывает:**
- Электроснабжение, Теплоснабжение, Горячее/Холодное водоснабжение, Газоснабжение
- Два типа: **Активные** (отключено прямо сейчас) и **Плановые** (запланировано сегодня)
- Количество затронутых домов по каждому району
- Исторические снимки — видно, что и когда отключали

### `bot topics`
Показать все поддерживаемые темы со статусом кэша.

```bash
bot topics
```

---

### `bot update`
Скачать/обновить данные из opendata.novo-sibirsk.ru.

```bash
bot update --all                  # все 10 тем
bot update --topic parking        # конкретная тема
bot update --topic schools --force  # принудительное обновление
```

---

### `bot serve`
Запустить HTTP API (FastAPI) на http://localhost:8000.

```bash
bot serve
# GET /topics
# GET /ask?q=сколько+парковок
# POST /update?topic=parking
```

## Подрайоны и микрорайоны

Бот распознаёт неформальные названия территорий и автоматически маппит их на родительский административный район.
Данные в открытых источниках хранятся на уровне района — при запросе по подрайону бот показывает данные по всему родительскому району и сообщает об этом.

### Советский район

| Что пишет пользователь | Подрайон | Маппится в |
|------------------------|----------|-----------|
| «в Академгородке», «Академгородок», «Academ», «в Академе» | Академгородок | Советский район |
| «на Шлюзе», «в Шлюзе», «Шлюз» | Шлюз | Советский район |
| «в Верхней зоне», «Верхняя зона» | Верхняя зона | Советский район |
| «мкр. Щ», «микрорайон Щ», «в Щ», просто «Щ» | мкр. "Щ" | Советский район |

Примеры запросов:

```bash
bot ask "аптеки в Академгородке"
# → фильтр: Советский район, подрайон=Академгородок
# → выводит все аптеки Советского района с примечанием

bot ask "школы на Шлюзе"
# → фильтр: Советский район, подрайон=Шлюз

bot ask "детские сады в мкр. Щ"
# → фильтр: Советский район, подрайон=мкр. "Щ"

bot ask "отключения электричества в Академгородке"
# → power_outages, район=Советский район, подрайон=Академгородок
```

> **Примечание для разработчика:** Подрайоны добавляются в `src/router.py` — список `_SUB_DISTRICTS`.
> Чтобы добавить новый подрайон, достаточно добавить строку с regex-паттерном, родительским районом и отображаемым названием.

## Экология и метеорология

Данные о качестве воздуха и погоде — в реальном времени с Open-Meteo и (опционально) CityAir.
TTL = 15 минут. История хранится 7 дней.

```bash
bot ecology update                       # загрузить актуальные данные
bot ecology status                       # AQI, PM2.5, погода по всем 10 районам
bot ecology status --district "Советский район"
bot ecology pdk                          # превышения ПДК WHO по PM2.5 за сегодня
bot ecology history                      # динамика за 7 дней
bot ecology history --days 3 --district "Центральный район"
```

**Что показывает:**
- PM2.5, PM10, NO2 — концентрация взвешенных частиц и диоксида азота (мкг/м³)
- AQI (European Air Quality Index) — сводный индекс качества воздуха
- Температура воздуха (°C), скорость ветра (м/с), влажность (%), давление (гПа)
- Превышения ПДК: PM2.5 > 35 мкг/м³ (стандарт ВОЗ) по районам за сутки
- Исторические снимки по дням с корреляцией ветра и PM2.5

**Источники данных:**

| Источник | TTL | Ключ | Данные |
|---|---|---|---|
| [Open-Meteo Air Quality](https://air-quality-api.open-meteo.com) | 15 мин | не нужен | PM2.5, PM10, NO2, AQI |
| [Open-Meteo Forecast](https://api.open-meteo.com) | 15 мин | не нужен | температура, ветер, давление, влажность |
| CityAir API | 15 мин | `CITYAIR_API_KEY` в `.env` | телеметрия физических датчиков |

> CityAir является опциональным источником. Без ключа бот работает только через Open-Meteo.

**Поддерживаемые запросы через `bot ask`:**

```bash
bot ask "какое качество воздуха в Новосибирске"
bot ask "где сейчас самый загрязненный воздух"
bot ask "превышение ПДК по PM2.5 сегодня"
bot ask "динамика PM2.5 в Советском районе за неделю"
bot ask "какая сейчас погода в центре"
bot ask "скорость ветра по районам"
bot ask "смог в Академгородке"
```

**Ключевые слова-триггеры:**

| Слово в запросе | Что запрашивает |
|---|---|
| «воздух», «загрязнен», «смог», «гарь» | Качество воздуха (AQI, PM) |
| «пыль», «частиц», «PM2.5», «PM10», «NO2» | Конкретные показатели |
| «экология», «экологическ» | Обзор качества воздуха |
| «ПДК», «превышен», «опасн», «PM2» | Превышения нормы (ECO_PDK) |
| «динамика», «история», «за неделю» | История по дням (ECO_HISTORY) |
| «погода», «температур», «ветер» | Метеоданные (ECO_STATUS) |
| «давлен», «влажност», «метеор» | Атмосферные показатели |

**Схема DuckDB (ТЗ §4):**

- `dim_stations` — справочник станций (10 районов, координаты, адреса)
- `fact_measurements` — журнал измерений (PM2.5, PM10, NO2, AQI, погода, timestamp)

## Архитектура

```
src/
  cli.py              — команды Click (ask / update / topics / serve / power / ecology)
  registry.py         — загрузка config/datasets.yaml
  fetcher.py          — HTTP-загрузка CSV с opendata.novo-sibirsk.ru
  parser.py           — нормализация CSV (encoding, delimiter)
  cache.py            — DuckDB: хранение данных opendata
  router.py           — маршрутизация запроса к теме (keywords + power + ecology)
  planner.py       — определение операции (count/topN/filter/group/power*)
  executor.py      — SQL-запросы к DuckDB + execute_power()
  renderer.py      — вывод результата (rich tables)
  power_scraper.py    — скрапинг 051.novo-sibirsk.ru (BeautifulSoup)
  power_cache.py      — DuckDB: power_outages, скользящее окно 7 дней
  ecology_fetcher.py  — ETL Open-Meteo + CityAir (качество воздуха + погода)
  ecology_cache.py    — DuckDB: dim_stations + fact_measurements, скользящее окно 7 дней

config/datasets.yaml  — реестр 10 тем opendata
DATA/
  raw/<topic>/     — сырые CSV-файлы
  cache.db         — DuckDB: topic_* + power_outages
  meta.json        — даты обновления и статистика
  logs/fetch.log   — лог HTTP-запросов
```

| Источник | Тип | Обновление |
|----------|-----|-----------|
