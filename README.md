# NSK OpenData Bot

CLI-агент и веб-интерфейс для работы с открытыми данными и системами жизнеобеспечения Новосибирска.

> **Источники данных:**
> - Открытые данные мэрии: [opendata.novo-sibirsk.ru](http://opendata.novo-sibirsk.ru) — парковки, школы, аптеки и др.
> - Отключения ЖКХ в реальном времени: [051.novo-sibirsk.ru](http://051.novo-sibirsk.ru)

---

## Как работает бот: архитектура Claude Agent SDK + SKILL.md

Бот построен на новой архитектуре **Claude Agent SDK** от Anthropic — фреймворка для создания интеллектуальных агентов поверх языковых моделей Claude.

## Нужна ли ИИ-модель для работы бота?

Текущий бот не использует никаких AI-моделей в работе.
Вся логика — правила:

router.py — regex + keyword matching (без нейросетей)
planner.py — regex-паттерны на русские слова
executor.py — SQL к DuckDB
power_scraper.py — HTML-парсинг
Бот работает локально, без API-ключей, без интернет-запросов к ИИ. Claude упомянут в роли инструмента разработчика (Claude Code читает SKILL.md чтобы помогать писать код), но не как часть рантайма бота.

### Роль файла `SKILL.md`

В корне проекта находится файл [`SKILL.md`](./SKILL.md) — это **семантическая конфигурация агента**. Когда Claude Code (или любой агент на базе Claude) получает запрос пользователя, он первым делом читает этот файл, чтобы понять:

- **Что умеет инструмент** — список команд, тем, операций
- **Как интерпретировать запросы** — примеры фраз, которые маппятся на конкретные действия
- **Каким тоном отвечать** — без переспрашивания, с максимальной лояльностью к качеству вопроса

Благодаря SKILL.md Claude не спрашивает «что именно вы имеете в виду?», а сразу пытается выполнить наиболее вероятную интерпретацию:

```
Пользователь:  «парковки»
Без SKILL.md:  «Уточните, что именно вы хотите узнать о парковках?»
С SKILL.md:    bot ask "парковки по районам"  → сразу выдаёт группировку
```

```
Пользователь:  «свет отключили в советском»
Без SKILL.md:  «Вы имеете в виду отключение электроэнергии в Советском районе?»
С SKILL.md:    bot ask → power_outages / POWER_STATUS / район=Советский
```

### Принцип «лояльного агента»

SKILL.md настраивает Claude на стратегию **best-effort interpretation** — сделать максимально полезный ответ на любой запрос, даже если он:
- написан с опечатками или сокращениями
- содержит только ключевые слова без контекста
- использует разговорный стиль вместо формальных команд
- не совпадает точно ни с одним примером из документации

Если запрос совсем непонятен, бот не выдаёт ошибку, а показывает список того, что он умеет, — так пользователь видит варианты и уточняет запрос самостоятельно.

### Схема работы

```
Запрос пользователя (русский текст)
        ↓
  Claude читает SKILL.md
  (понимает контекст и намерение)
        ↓
  router.py — маршрутизация к теме
  (keyword matching + stem prefixes)
        ↓
  planner.py — определение операции
  (COUNT / GROUP / TOP_N / FILTER / POWER_*)
        ↓
  executor.py — SQL к DuckDB
        ↓
  renderer / API — результат
```

### Как расширить бота через SKILL.md

Чтобы добавить новые возможности, которые Claude будет понимать «из коробки», достаточно описать их в SKILL.md — без изменения кода:

```markdown
## Новая возможность

bot ask "медицинские учреждения в Калининском районе"
bot ask "поликлиники по районам"
```

После добавления описания Claude сможет корректно интерпретировать такие запросы и вызывать нужные команды — даже если пользователь напишет «где врачи в калинике».

## Быстрый старт (локально)

### 1. Требования

- **macOS / Linux** (на Windows работает через WSL)
- **Python 3.11+**
- Интернет-соединение для первичной загрузки данных

Проверьте версию Python:
```bash
python3 --version   # должно быть 3.11 или выше
```

### 2. Установка

```bash
# Скачайте проект (если ещё не сделали)
git clone https://github.com/ВАШ_ЛОГИН/nsk-opendata-bot.git
cd nsk-opendata-bot

# Установите бот (один раз)
pip install -e .
```

### 3. Загрузка данных

```bash
# Загрузить все открытые данные (~1–2 минуты, ~4 МБ)
bot update --all

# Загрузить данные об отключениях ЖКХ (мгновенно)
bot power update
```

### 4. Запуск

**Через браузер (рекомендуется):**
```bash
bot serve
# Открыть в браузере: http://127.0.0.1:8000
```

**Через командную строку:**
```bash
bot ask "сколько школ в Ленинском районе"
bot ask "плановые отключения электричества"
bot topics   # список всех тем
```

## Запуск без IDE (macOS)

В папке проекта есть файл **`Запустить бот.command`**.

1. Откройте **Finder** → папка `nsk-opendata-bot`
2. Двойной клик на **`Запустить бот.command`**
3. При первом запуске macOS спросит разрешение → нажмите **«Открыть»**
4. Откроется терминал, через 2–3 секунды браузер с ботом
5. Браузер можно закрыть — бот продолжает работать
6. Чтобы **остановить** бот — закройте окно терминала

> Если macOS заблокировала файл: `System Settings → Privacy & Security → Allow`

**Автозапуск при включении ноутбука:**
```bash
# Выполнить один раз в терминале:
cat > ~/Library/LaunchAgents/ru.nsk.opendata.bot.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0"><dict>
  <key>Label</key><string>ru.nsk.opendata.bot</string>
  <key>ProgramArguments</key><array>
    <string>/bin/sh</string><string>-c</string>
    <string>cd ~/nsk-opendata-bot && bot serve</string>
  </array>
  <key>RunAtLoad</key><true/>
  <key>KeepAlive</key><true/>
</dict></plist>
EOF
launchctl load ~/Library/LaunchAgents/ru.nsk.opendata.bot.plist
```

После этого бот доступен на `http://127.0.0.1:8000` сразу после входа в систему.


## Публичный сайт (деплой в облако)

> ⚠️ **Vercel не подходит** для этого проекта: Vercel — serverless-платформа без постоянного хранилища файлов. DuckDB требует запись на диск, а сервер должен работать непрерывно.

### Рекомендуется: Railway (бесплатно, без кредитной карты)

**Railway** — оптимальный выбор: поддерживает Python, FastAPI, постоянный диск, бесплатный тариф.

#### 1. Создайте `Procfile` в корне проекта:
```
web: bot update --all && bot power update && bot serve --host 0.0.0.0 --port $PORT
```

#### 2. Создайте `runtime.txt`:
```
python-3.11
```

#### 3. Задеплойте:

1. Зайдите на [railway.app](https://railway.app) → **«New Project»**
2. Выберите **«Deploy from GitHub repo»**
3. Подключите репозиторий `nsk-opendata-bot`
4. Railway автоматически обнаружит `pyproject.toml` и запустит сборку
5. В разделе **Settings → Networking** нажмите **«Generate Domain»**
6. Сайт будет доступен по адресу вида `nsk-opendata-bot.up.railway.app`

> Бесплатный тариф Railway: 500 часов/месяц (≈ 20 дней непрерывной работы).
> Для постоянной работы нужен тариф Hobby ($5/мес).

---

### Альтернатива: Render (бесплатный тариф с ограничениями)

1. Зайдите на [render.com](https://render.com) → **«New Web Service»**
2. Подключите GitHub-репозиторий
3. Настройки:
   - **Runtime:** Python 3
   - **Build command:** `pip install -e .`
   - **Start command:** `bot update --all && bot serve --host 0.0.0.0 --port $PORT`
4. **Add Disk** (в разделе Storage) — для сохранения кэша DuckDB
   - Mount path: `/root/nsk-opendata-bot/data`
   - Size: 1 GB

> Бесплатный тариф Render: сервис «засыпает» после 15 мин неактивности (первый запрос будет медленным).
