# NSK OpenData Bot · v1.0.5


CLI-агент и веб-интерфейс для работы с открытыми данными и системами жизнеобеспечения Новосибирска.
Доступен по ссылке: https://nsk-opendata-bot.up.railway.app/

> **Источники данных:**
> - Открытые данные мэрии: [opendata.novo-sibirsk.ru](http://opendata.novo-sibirsk.ru) — парковки, школы, аптеки, остановки и др.
> - Отключения ЖКХ в реальном времени: [051.novo-sibirsk.ru](http://051.novo-sibirsk.ru)
> - Экология и погода: [Open-Meteo](https://open-meteo.com) — качество воздуха (AQI, PM2.5, PM10, NO₂) и метеорология · обновление раз в 15 мин
> - Камеры фиксации нарушений: [OpenStreetMap](https://www.openstreetmap.org) via Overpass API — стационарные радары ПДД · TTL 7 дней · лицензия ODbL
> - Маршруты транспорта: база остановок opendata.novo-sibirsk.ru — нет внешних API, ключи не нужны
> - Индекс пробок: аналитический расчёт (время, праздники, погода) — без реального трафика

## Как работает бот: rule-based Python без нейросетей

Бот — чистый Python без ИИ в рантайме. Нейросеть не используется ни для одного запроса пользователя. Вся логика — детерминированные правила:

- `router.py` — keyword matching + stem prefixes (без нейросетей)
- `planner.py` — regex-паттерны на русские слова → операция
- `executor.py` — SQL к DuckDB / HTTP к Open-Meteo
- `power_scraper.py` — HTML-парсинг 051.novo-sibirsk.ru
- Бот работает локально, без API-ключей, без интернет-запросов к ИИ

### Схема работы

```
Запрос пользователя (русский текст)
        ↓
  router.py — маршрутизация к теме
  (keyword matching + stem prefixes, без нейросетей)
        ↓
  planner.py — определение операции
  (COUNT / GROUP / TOP_N / FILTER / POWER_* / ECO_*)
        ↓
  executor.py — SQL к DuckDB / HTTP к Open-Meteo
        ↓
  renderer / API — результат
```

Если запрос совсем не распознан — бот показывает список доступных тем, а не сообщение об ошибке.

[`SKILL.md`](./SKILL.md) — документация проекта для **разработчика**. Её читает Claude Code (AI-инструмент разработки), чтобы понимать устройство бота и помогать писать код. К работе бота в продакшне этот файл никакого отношения не имеет.

## Роли пользователей (веб-интерфейс)

Веб-интерфейс поддерживает три режима работы. Переключатель роли находится **в правом верхнем углу** — рядом с кнопкой «API Docs». Роль сохраняется в браузере между сессиями.

### 👤 Горожанин (по умолчанию)

Режим для жителей города. Показывает подсказки для повседневных запросов:

- Отключения горячей воды / электричества / отопления в конкретном районе
- Ближайшие библиотеки, аптеки, школы — с кнопкой «📍 На карте» (синие чипы)
- Качество воздуха и погода по подрайонам (Академгородок, Кольцово)
- Риски для здоровья и безопасности
- **Маршруты транспорта** — «как проехать из Академгородка в Дзержинский район»
- **Пробки** — «пробки сейчас» → аналитический индекс с советом и факторами нагрузки

Чипы, помеченные 📍 (синие), возвращают список объектов с адресами — после результата появляется кнопка «Показать на карте 2GIS».
Для карт нужен ключ: 2ea1a391-110d-4e04-a73e-32f6425a9092

### 🏙️ Служащий умного города

Режим для сотрудников мэрии и городских служб. Помимо тематических подсказок (статистика по районам, аналитика ЖКХ, история отключений) — показывает **панель диспетчера**, которая автоматически загружается при входе в режим:

| Элемент | Данные |
|---|---|
| 🔴 Аварийных домов | Сумма всех типов ЖКХ по последнему скрапу |
| 🟡 Плановых домов | Запланированные отключения |
| 🍃 AQI (средний) | Средний индекс качества воздуха по всем районам |
| ⚠️ Активных рисков | Число активных угроз по ecology_cache |
| Таблица аварий | Топ-5 районов с аварийными отключениями |
| AQI-чипы | Кликабельные метки AQI по каждому району |

Все четыре KPI-плитки кликабельны — переходят к соответствующим запросам.

#### Фильтр по районам (новое в v1.0.4)

Служащий может закрепить **район ответственности** — тогда панель показывает только данные по этому району. Настройка доступна в двух местах:

**1. При выборе роли** — двухуровневый дропдаун:
- Кнопка роли → список ролей
- «Служащий умного города `›`» → пикер районов (10 районов + Академгородок + Кольцово + «Весь город»)
- После выбора кнопка показывает короткое название: `🏙️ Советский`

**2. Прямо в панели** — горизонтальная полоска тегов под заголовком:
```
Район: 🌆 Весь город | Дзержинский | Железнодорожный | … | Академгородок | Кольцово
```
Клик по тегу мгновенно перефильтровывает данные без перезагрузки страницы. Выбранный район сохраняется в браузере между сессиями.

**Что фильтруется:**
- KPI-плитки (аварийные/плановые дома, AQI) — пересчитываются по данным района
- Таблица аварийных отключений — только строки района, до 10 записей
- AQI-чипы — только чип выбранного района
- Виджет остановок — нужный район поднят наверх и выделен синим
- Все onclick-запросы — автоматически добавляют контекст: «аварийные отключения ЖКХ в Советском районе»

> Подрайон «Академгородок» матчится через родительский «Советский район» — данные отображаются корректно несмотря на отсутствие отдельной строки в источниках.

### 🏛️ Мэр

Режим для руководителей. Показывает **сводную KPI-панель** с крупными числами и статус-индикатором:

| Статус | Условие |
|---|---|
| 🟢 Штатный режим | Нет активных рисков, аварийных домов < 500 |
| 🟡 Требует внимания | Есть активные риски |
| 🔴 Нештатная ситуация | Аварийных домов > 500 или рисков > 2 |

Все четыре KPI-плитки кликабельны — переходят к соответствующим запросам. Карточки активных угроз отображаются ниже KPI.

## Фильтр по районам для диспетчера (новое в v1.0.4)

**Краткая схема работы:**

```
Выбор роли → пикер района → сохранение в localStorage
      ↓

      ↓

```

## Маршруты общественного транспорта (новое в v1.0.3)

Бот умеет находить **общие маршруты наземного транспорта** между любыми двумя районами Новосибирска. Данные берутся из открытых данных мэрии (тема `stops`) — без внешних API и без ключей.

### Как работает

1. Из запроса извлекаются районы «откуда» и «куда» (поддерживаются полные названия и подрайоны — Академгородок, Шлюз и др.)
2. В базе остановок (таблица `topic_stops`) выбираются все остановки в каждом районе
3. Из поля `Marshryt` парсятся номера маршрутов (автобусы, троллейбусы, трамваи, маршрутки)
4. Находится пересечение — маршруты, которые обслуживают оба района одновременно
5. Для каждого маршрута показываются примеры остановок в точке отправления и назначения

### Примеры запросов

| Запрос | Что происходит |
|---|---|
| `как проехать из Академгородка в Дзержинский район` | 8 общих маршрутов с примерами остановок |
| `как добраться из Советского района в Центральный` | Маршруты через весь город |
| `маршрут из Кировского в Калининский район` | Пересадочные маршруты |

Результат всегда включает **ссылку на 2ГИС** для построения точного пошагового маршрута.

### Доступные эндпоинты

- `GET /ask?q=как проехать из X в Y` — через NLP-конвейер (автоматическое распознавание районов)
- `GET /transit?from_district=Советский район&to_district=Дзержинский район` — прямой вызов
- `GET /transit/districts` — количество остановок по районам (для панели диспетчера)

> ⚠️ Данные из открытых данных мэрии могут не включать все коммерческие маршруты. Для точной навигации используйте 2ГИС или Яндекс.Транспорт.

## Индекс дорожной нагрузки (новое в v1.0.3)

Бот рассчитывает **аналитический индекс пробок** по шкале 0–10 без использования реального трафика — на основе исторических паттернов, праздничного календаря и текущей погоды.

> ⚠️ Индекс аналитический, не использует данные GPS-трекеров или сенсоров дорожного движения. Для Новосибирска открытых источников реального трафика нет.

### Факторы расчёта

| Фактор | Вклад | Примеры |
|---|---|---|
| **Время суток** | основа | Утренний час пик 7:30–9:30 = +5.5, ночь = +0.3 |
| **День недели** | ×0.45 в выходные | Понедельник = +0.5 (эффект «сонного понедельника») |
| **Официальные праздники** | снижение | Нерабочие дни, мосты, предпраздничные |
| **Сезонные события** | разовые | 1 Сентября = +2.0, канун Нового года = +2.0 |
| **Погода: снег** | +1.0 … +2.5 | Первый осенний снег (окт–ноя): +1.5 дополнительно |
| **Погода: дождь** | +0.5 … +1.0 | Сильный дождь / ливень |
| **Погода: гололёд** | +2.5 | Ледяной дождь (WMO 56, 57, 66, 67) |
| **Погода: гроза** | +0.5 | Видимость снижена |
| **Мороз ниже −20°C** | +0.5 | Замедленные реакции, скользко |
| **Сильный ветер > 15 м/с** | +0.5 | Боковой снос, видимость |

### Праздничная база 2025–2027

Встроена полная база официальных нерабочих дней, переносов и предпраздничных (укороченных) дней РФ на три года:

- **holiday** — нерабочий день
- **bridge** — перенесённый рабочий → нерабочий («мост»)
- **pre** — предпраздничный, укороченный на 1 час
- **post_holiday** — первый рабочий после длинных каникул (+0.8 к нагрузке)

### Шкала индекса

| Значение | Уровень | Рекомендация горожанину |
|---|---|---|
| 0.0 – 1.9 | 🟢 Свободно | Отличное время для поездки |
| 2.0 – 3.4 | 🟡 Умеренно | Небольшие задержки возможны |
| 3.5 – 4.9 | 🟠 Затруднено | Заложите +15 мин |
| 5.0 – 6.4 | 🔴 Сложно | Рассмотрите общественный транспорт |
| 6.5 – 8.4 | 🔴 Очень сложно | Объезд / метро / самокат |
| 8.5 – 10.0 | ⛔ Коллапс | Останьтесь дома или используйте транспорт |

### Примеры запросов

```
пробки сейчас
трафик в городе
загруженность дорог
час пик сегодня
```

### Эндпоинт

```
GET /traffic-index
```

Ответ включает: индекс, уровень, факторы нагрузки, совет горожанину, совет администрации, ближайший пик.

## Экология и погода (новое в v1.0.1)

Бот получает данные о **качестве воздуха и погоде** по всем 10 районам Новосибирска через API [Open-Meteo](https://open-meteo.com) каждые 15 минут.

### Доступные запросы

| Запрос | Операция | Описание |
|---|---|---|
| `качество воздуха в городе` | ECO_STATUS | Сводка по всем районам: AQI, PM2.5, температура, ветер |
| `погода в Советском районе` | ECO_STATUS | Данные по конкретному району |
| `погода в Академгородке` | ECO_STATUS | Поддержка подрайонов (Шлюз, Верхняя зона, мкр. «Щ») |
| `превышение ПДК по PM2.5` | ECO_PDK | Районы с PM2.5 > 35 мкг/м³ (норма ВОЗ) |
| `динамика воздуха за неделю` | ECO_HISTORY | История по дням (до 30 дней) |
| `риски для жизни в городе` | ECO_RISKS | Прескриптивная аналитика по сценариям |

### Прескриптивная аналитика (умный город)

Запрос «**риски для жизни в городе**» запускает rule-based движок, который проверяет текущие данные и формирует карточки рисков:

| Сценарий | Триггер | Рекомендации |
|---|---|---|
| 🌫️ Экологическая ловушка | Ветер < 1.5 м/с + PM2.5 > 20 мкг/м³ | Горожанам: не открывайте окна. Службам: режим НМУ |
| ☢️ Превышение нормы ВОЗ | PM2.5 > 35 мкг/м³ | Горожанам: маска FFP2. Службам: оповещение населения |
| 🧊 Риск гололёда | Температура −3..+2°C | Горожанам: +20 мин на маршрут. Службам: пескоразбрасыватели |
| ⚠️ Температурный шок | Суточное падение ≥ 15°C | Горожанам: многослойная одежда. Службам: аварийный режим ЖКХ |
| 🥶 Экстремальный холод | Температура < −20°C | Горожанам: ограничить время на улице. Службам: пункты обогрева |

### Индексы жизнедеятельности

На странице «Качество воздуха» отображаются три индекса (0–10):

- **🚗 Индекс водителя** — опасность на дорогах (гололёд, экстремальный холод, сильный ветер)
- **🚶 Индекс прогулки** — комфорт на улице (PM2.5, AQI, температура)
- **🏗️ Коммунальное напряжение** — риск инцидентов в ЖКХ (мороз, безветрие + смог)

### Накопление исторических данных

- Сырые измерения хранятся в скользящем окне **30 дней** (таблица `fact_measurements`)
- Ежесуточные агрегаты сохраняются в **постоянный архив** `ecology_daily_archive` (до 365 дней)
- При запросе истории за > 30 дней используются данные из архива
- Данные накапливаются автоматически при каждом обновлении (раз в 15 мин при работающем боте)

## Интерактивная карта 2GIS (v1.0.1)

Если задан ключ 2GIS API (`TWOGIS_API_KEY` в env или `data/api_keys.json`), в шапке веб-интерфейса появляется значок **🗺 2GIS**, а результаты запросов со списками объектов получают кнопку **«📍 Показать на карте»**.

### Какие запросы поддерживают карту

Карта доступна для операций **FILTER** и **TOP_N** — то есть запросов, которые возвращают список объектов с адресами. В интерфейсе такие подсказки подсвечены синим и помечены значком 📍.

| Запрос | Операция | Объекты на карте |
|---|---|---|
| `топ-5 аптек` | TOP_N | Аптеки (до 5 точек) |
| `аптеки в Центральном районе` | FILTER | Аптеки выбранного района |
| `аптеки в Ленинском районе` | FILTER | Аптеки выбранного района |
| `библиотеки для детей` | FILTER | Детские библиотеки |
| `библиотеки города` | FILTER | Все библиотеки Новосибирска |
| `школы в Советском районе` | FILTER | Школы района |
| `детские спортивные организации` | FILTER | Спортивные клубы и секции |
| `культурные организации` | FILTER | Музеи, театры, центры культуры |
| `детские сады в Кировском районе` | FILTER | Детские сады района |

Карта **не работает** для COUNT («сколько...») и GROUP («по районам») — это агрегированные цифры без отдельных адресов.

### Как работает геокодирование

1. Пользователь нажимает кнопку **«📍 Показать на карте»** под таблицей результатов
2. Бот делает повторный запрос с параметром `with_coords=true`
3. Каждый адрес преобразуется в координаты через **2GIS Geocoder API**
4. Результат кешируется в DuckDB — повторные запросы мгновенны (без API-вызова)
5. На карте **MapGL JS** отображаются маркеры; скрипт карты загружается лениво при первом использовании

Первое геокодирование: ~0.5–2 с на адрес через API. Из кеша: мгновенно.
Максимум — 50 объектов за один запрос (`max_rows=50` в `geocode_rows()`).

### Требования

- Ключ 2GIS API: получить бесплатно на [platform.2gis.ru](https://platform.2gis.ru)
- Сохранить через `POST /twogis/key?key=<ключ>` в веб-интерфейсе API или в `data/api_keys.json`
- Интернет-соединение (для загрузки MapGL JS и первичного геокодирования)

### Ограничения лицензии 2GIS

Согласно [law.2gis.ru/api-rules](https://law.2gis.ru/api-rules): кеширование результатов геокодирования разрешено. Данные трафика и маршруты общественного транспорта — только pass-through, без хранения.

## 🏗️ Строительство в Новосибирске (новое в v1.0.5)

Бот умеет работать с двумя официальными реестрами строительных объектов мэрии Новосибирска:

| Реестр | Датасет | Записей | Ключевые поля |
|---|---|---|---|
| Разрешения на строительство | [ID 124](http://opendata.novo-sibirsk.ru/pass.aspx?ID=124) | ~5 942 | Номер, дата, застройщик, объект, адрес, кадастровый № |
| Ввод в эксплуатацию | [ID 125](http://opendata.novo-sibirsk.ru/pass.aspx?ID=125) | ~1 935 | Номер, дата, застройщик, объект, район, адрес, кадастровый № |

**Активные стройки** вычисляются как разность множеств: объекты из реестра 124, кадастровый номер которых **отсутствует** в реестре 125. То есть: получено разрешение, но ввода в эксплуатацию ещё не было.

### Поддерживаемые запросы

| Запрос | Операция | Описание |
|---|---|---|
| `активные стройки в Калининском районе` | `CONSTRUCTION_ACTIVE` | Список объектов с разрешением, без ввода в эксплуатацию |
| `строящиеся объекты` | `CONSTRUCTION_ACTIVE` | Активные стройки без фильтра по району |
| `разрешения на строительство` | `CONSTRUCTION_ACTIVE` | Все разрешения (с фильтром по застройщику, адресу) |
| `сколько активных строек` | `CONSTRUCTION_COUNT` | Суммарное количество активных объектов |
| `стройки по районам` | `CONSTRUCTION_GROUP` | Количество активных строек по каждому из 10 районов |
| `ввод в эксплуатацию в Советском районе` | `CONSTRUCTION_COMMISSIONED` | Введённые в эксплуатацию объекты |
| `сколько введено в эксплуатацию` | `CONSTRUCTION_COUNT` | Счётчик сданных объектов |
| `ввод в эксплуатацию по районам` | `CONSTRUCTION_GROUP` | Распределение сданных объектов по районам |

### CLI-команды

```bash
# Загрузить/обновить данные (TTL 24 часа)
bot construction update

# Показать статистику
bot construction status

# Список активных строек (с фильтром по району)
bot construction active --district "Калининский"
```

### Как работает определение района

Датасет 124 не содержит явного поля «район» — он закодирован в адресе (например: «Калининский район, пер. Турухановский, 14»). Бот извлекает район через SQL `CASE … ILIKE` для всех 10 административных районов. Датасет 125 содержит явное поле `Raion`.

### Обновление данных

Данные хранятся в DuckDB, TTL = 24 часа. Обновление: `bot construction update` или автоматически при `bot update --all`.

## Публичный сайт (деплой в облако)

> ⚠️ **Vercel не подходит** для этого проекта: Vercel — serverless-платформа без постоянного хранилища файлов. DuckDB требует запись на диск, а сервер должен работать непрерывно.

### Рекомендуется: Railway (бесплатно, без кредитной карты)

**Railway** — оптимальный выбор: поддерживает Python, FastAPI, постоянный диск, бесплатный тариф.

#### 1. Создайте `Procfile` в корне проекта:
```
web: bot update --all && bot power update && bot serve --host 0.0.0.0 --port $PORT
```

#### 2. Создайте `runtime.txt`:
```
python-3.11
```

#### 3. Задеплойте:

1. Зайдите на [railway.app](https://railway.app) → **«New Project»**
2. Выберите **«Deploy from GitHub repo»**
3. Подключите репозиторий `nsk-opendata-bot`
4. Railway автоматически обнаружит `pyproject.toml` и запустит сборку
5. В разделе **Settings → Networking** нажмите **«Generate Domain»**
6. Сайт будет доступен по адресу вида `nsk-opendata-bot.up.railway.app`

> Бесплатный тариф Railway: 500 часов/месяц (≈ 20 дней непрерывной работы).
> Для постоянной работы нужен тариф Hobby ($5/мес).

---

### Альтернатива: Render (бесплатный тариф с ограничениями)

1. Зайдите на [render.com](https://render.com) → **«New Web Service»**
2. Подключите GitHub-репозиторий
3. Настройки:
   - **Runtime:** Python 3
   - **Build command:** `pip install -e .`
   - **Start command:** `bot update --all && bot serve --host 0.0.0.0 --port $PORT`
4. **Add Disk** (в разделе Storage) — для сохранения кэша DuckDB и истории экологии
   - Mount path: `/root/nsk-opendata-bot/data`
   - Size: 1 GB

> Бесплатный тариф Render: сервис «засыпает» после 15 мин неактивности (первый запрос будет медленным).
