Что такое NSK OpenData Bot

Бот — это полноценный интеллектуальный агент, который принимает свободный текст на русском языке и превращает его в точный SQL-запрос к городским данным. Никакой ручной настройки форм, фильтров, кнопок — пользователь просто пишет «сколько школ в Ленинском районе» и мгновенно получает ответ.

Ключевая идея: семантика как движок. Вместо того чтобы учить пользователей языку системы, система учится языку пользователя. Это достигается через комбинацию правил (router.py + planner.py) и семантической конфигурации (SKILL.md).

Архитектура системы

Запрос проходит через четыре слоя: маршрутизация → планирование → выполнение → рендеринг. Каждый слой имеет единственную ответственность и легко тестируется независимо (79 unit-тестов).

Семантическое ядро: как работает маршрутизатор

Маршрутизатор (router.py) — это взвешенная система сопоставления смыслов, не просто поиск по ключевым словам. Он учитывает морфологию русского языка через стем-префиксы и умеет разрешать семантические конфликты (например, «парк» ≠ «парковка»).

🔤 Стем-префиксы вместо словарей

Вместо хранения всех словоформ ключевое слово записывается как корень: спортивн → совпадает с «спортивные», «спортивной», «спортивных». Паттерн (?<![а-яё])stem гарантирует совпадение только с началом слова.

⚖️ Взвешенная оценка уверенности

Каждое совпадающее ключевое слово даёт очки: len(kw_parts)^1.5. Длинные фразы («парк культур», «спортивн организ») ценятся больше коротких корней. Итоговая уверенность нормируется к [0, 1].

🎯 Дизамбигуация омонимов

«Парков» — это генитив от «парк» (зелёная зона) или часть «парковок» (машины)? Два regex-паттерна разрешают конфликт: parking_signals удаляют «парки» при наличии «стоянк», «парковк»; park_exact добавляет «парки» для голого слова «парк» с проверкой конца слова.

🗺️ Извлечение сущностей

Из запроса автоматически извлекаются: район (Советский, Центральный…), улица (ул. Ленина, проспект Карла Маркса), лимит (топ-5, первые 10), год (за 2024 год), аудитория (детские / взрослые). 

# Пример: как оценивается запрос «спортивные площадки в Советском районе»

route("спортивные площадки в Советском районе")

# router.py проверяет каждое ключевое слово из datasets.yaml:
# "спортплощадк"   → нет совпадения с "спортивные площадки"
# "спортивн площадк" → "спортивн" ✓ + "площадк" ✓ → score += 2^1.5 = 2.83
# confidence = min(1.0, 2.83 / 6 * 3) = 0.94

# planner.py:
# "в Советском районе" → district = "Советский район"
# нет "сколько", "топ", "по районам" → operation = FILTER

# executor.py строит SQL:
SELECT "Type", "District", "Street", "House", "VidSport", "Phone", "Comment"
FROM sport_grounds
WHERE ("District" ILIKE '%Советский%')
ORDER BY "District", "Type"
LIMIT 20

SKILL.md — конфигурация агента для Claude

В корне проекта есть файл SKILL.md — это семантическая конфигурация агента. Когда Claude Code (или любой агент на базе Claude) получает запрос пользователя, он сначала читает этот файл, чтобы понять контекст, команды и тон ответов.

💡 Ключевая идея SKILL.md

Вместо того чтобы программировать поведение агента в коде — оно описывается на естественном языке в Markdown-файле. Добавить новую тему или команду = написать несколько строк в SKILL.md, без изменения Python-кода.

Принцип «лояльного агента»: Best-effort interpretation
Без SKILL.md
«парковки»
«Уточните, что именно вы хотите узнать о парковках?»
С SKILL.md
«парковки»
→ parking / GROUP / по районам — сразу выдаёт группировку
Без SKILL.md
«свет отключили в советском»
«Вы имеете в виду отключение электроэнергии?»
С SKILL.md
«свет отключили в советском»
→ power_outages / POWER_STATUS / Советский
# Фрагмент SKILL.md — так выглядит семантическая конфигурация агента

## Принципы интерпретации

- Всегда пытаться выполнить наиболее вероятную интерпретацию
- Не спрашивать уточнения — выполнять с разумными допущениями
- При неопределённости — показывать список доступных тем

Почему это красиво: для разработчиков и заказчиков

🧩
Минимум зависимостей

Python + DuckDB + FastAPI + BeautifulSoup. Никакого Spark, Kafka, Elasticsearch или внешних векторных БД. Запускается на любом компьютере с 4 ГБ RAM.

⚡
Мгновенные ответы

DuckDB выполняет аналитические SQL-запросы к CSV в памяти за миллисекунды. Кэш на диске — данные загружаются один раз, используются тысячи раз.

🔧
Легко расширять

Добавить новую тему — добавить секцию в datasets.yaml с URL данных и ключевыми словами. Добавить смысл — написать строку в SKILL.md. Никакого обучения модели.

🧪
Полное тестовое покрытие

79 unit-тестов покрывают маршрутизатор, планировщик, извлечение сущностей и дизамбигуацию. Каждый PR проверяется автоматически — регрессии исключены.

🤖
Гибрид: правила + ИИ

Без LLM в runtime — быстро и предсказуемо. Опционально: подключите Claude API, GigaChat или Ollama для генерации ответов в свободной форме поверх структурированных данных.

🏙️
Готовый шаблон для городов

Архитектура универсальна: замените URL источников данных и ключевые слова — получите аналогичного бота для любого другого города с порталом открытых данных.

Структура проекта

nsk-opendata-bot/
├── src/
│   ├── router.py        # Семантическая маршрутизация запросов
│   ├── planner.py       # COUNT / GROUP / TOP_N / FILTER / POWER_*
│   ├── executor.py      # SQL-запросы к DuckDB
│   ├── power_scraper.py # Парсер 051.novo-sibirsk.ru (отключения ЖКХ)
│   ├── power_cache.py   # Скользящее окно 7 дней, TTL 30 мин
│   ├── api.py           # FastAPI: /ask, /topics, /update, /docs, /info
│   ├── cli.py           # CLI: bot ask / bot topics / bot power
│   ├── fetcher.py       # Загрузка CSV с opendata (utf-8-sig / cp1251)
│   ├── cache.py         # DuckDB-кэш с TTL
│   ├── renderer.py      # Rich-рендеринг в терминале
│   └── static/
│       ├── index.html   # Веб-интерфейс (Tailwind + Vanilla JS)
│       ├── info.html    # Эта страница
│       └── ...
├── config/
│   └── datasets.yaml    # Реестр тем: URL, ключевые слова, поля
├── tests/               # 79 unit-тестов
├── SKILL.md             # Семантическая конфигурация для Claude Agent SDK
├── pyproject.toml       # Точки входа: bot, bot serve, ...
└── Запустить бот.command # Лаунчер для macOS (двойной клик)
