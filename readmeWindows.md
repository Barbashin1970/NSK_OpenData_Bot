# Установка NSK OpenData Bot на Windows

Пошаговое руководство для запуска проекта на ноутбуке с Windows 10/11.

---

## 🔧 Требования

- **Windows 10 / 11** (x64)
- **Python 3.11+** (обязательно)
- **pip** (идёт с Python)
- **Git** (опционально, для истории коммитов)
- **Интернет** (для загрузки зависимостей)

---

## 📥 Шаг 1: Установка Python

### 1.1 Скачайте Python

Перейдите на [python.org](https://www.python.org/downloads/) и скачайте **Python 3.11 или выше** для Windows (рекомендуется **3.12 LTS**).

### 1.2 Установите Python

При запуске установщика **ВАЖНО**:

1. ✅ **Поставьте галку** `Add Python 3.X to PATH`
2. ✅ **Выберите** `Install for all users` (если есть такая опция)
3. Нажмите `Install Now`

![image](https://user-images.githubusercontent.com/...) <!-- При желании добавить скриншот -->

### 1.3 Проверьте установку

Откройте **Command Prompt** (Пуск → `cmd`) и выполните:

```cmd
python --version
pip --version
```

Должны вывести версии Python 3.11+ и pip.

---

## 📦 Шаг 2: Git (опционально)

Если хотите видеть историю коммитов в приложении, установите Git:

1. Скачайте с [git-scm.com](https://git-scm.com/)
2. При установке можно оставить параметры по умолчанию
3. Проверьте: `git --version` в Command Prompt

Если Git не установлен — приложение всё равно будет работать, просто без показа истории коммитов.

---

## 💾 Шаг 3: Скачайте проект

### Вариант 1: Через Git (если установлен)

```cmd
git clone https://github.com/Barbashin1970/NSK_OpenData_Bot.git
cd NSK_OpenData_Bot
```

### Вариант 2: Скачать ZIP (без Git)

1. Откройте [github.com/Barbashin1970/NSK_OpenData_Bot](https://github.com/Barbashin1970/NSK_OpenData_Bot)
2. Нажмите зелёную кнопку **Code** → **Download ZIP**
3. Распакуйте архив в удобное место, например: `C:\Users\YourName\NSK_OpenData_Bot`
4. В Command Prompt перейдите в эту папку:

```cmd
cd C:\Users\YourName\NSK_OpenData_Bot
```

---

## ⚙️ Шаг 4: Установка зависимостей

В Command Prompt (в папке проекта) выполните:

```cmd
pip install -e .
```

Это установит все необходимые библиотеки из `pyproject.toml`:
- `click` — CLI фреймворк
- `duckdb` — встроенная база данных
- `fastapi` + `uvicorn` — веб-сервер
- `requests`, `beautifulsoup4` — парсинг и HTTP
- И остальные...

Установка займёт 1–2 минуты.

---

## ▶️ Шаг 5: Запуск приложения

### 📌 Способ 1: Двойной клик на `startwindows.bat` (рекомендуется)

1. Откройте папку проекта в **File Explorer** (Проводнике)
2. Найдите файл **`startwindows.bat`**
3. **Двойной клик** на нём

**Что произойдёт:**
- ✓ Проверятся все зависимости (Python, pip, git)
- ✓ Установятся недостающие пакеты
- ✓ Запустится локальный сервер на `http://127.0.0.1:8000`
- ✓ Откроется браузер с главной страницей
- ✓ При закрытии окна сервер остановится

**Оставьте окно Command Prompt открытым** — при его закрытии сервер остановится.

### Способ 2: Команда в Command Prompt

Если батник не сработал, выполните вручную:

```cmd
bot serve
```

Затем откройте браузер: http://127.0.0.1:8000

## 🚨 Решение проблем

### ❌ "Python не найден"

**Проблема:** При запуске батника пишет, что Python не найден.

**Решение:**
1. Переустановите Python (убедитесь, что выбрана опция **Add Python to PATH**)
2. После переустановки перезагрузитесь
3. Откройте новый Command Prompt и попробуйте: `python --version`

### ❌ "pip install -e . failed"

**Проблема:** При установке зависимостей ошибка.

**Решение:**
1. Обновите pip:
   ```cmd
   python -m pip install --upgrade pip
   ```
2. Попробуйте установку снова:
   ```cmd
   pip install -e .
   ```

### ❌ "Порт 8000 занят"

**Проблема:** `Address already in use :8000`

**Решение:**
Батник автоматически останавливает старый процесс. Если не помогло:

```cmd
netstat -ano | findstr :8000
taskkill /PID <PID> /F
```

где `<PID>` — номер процесса из вывода выше.

### ❌ "Браузер не открывается"

**Проблема:** Сервер запустился, но браузер не открылся.

**Решение:**
Откройте браузер вручную и перейдите на: http://127.0.0.1:8000

### ⚠️ "Ошибка подключения к базе данных"

**Проблема:** DuckDB ошибка при запуске.

**Решение:**
1. Убедитесь, что в папке проекта есть подпапка `data/`
2. Если нет, создайте её:
   ```cmd
   mkdir data
   ```
3. Перезагрузите страницу браузера

## 📝 Полезные команды

```cmd
bot serve
```

### Проверка доступных команд CLI

```cmd
bot --help
```

Логи сохраняются в `data/logs/fetch.log`:

```cmd

```

## 📱 Для мобильного устройства в одной сети

Если вы хотите открыть приложение на смартфоне в одной локальной сети с ноутбуком:

1. Узнайте IP-адрес компьютера:
   ```cmd
   ipconfig | findstr "IPv4"
   ```
   Ищите строку типа: `IPv4 Address . . . . . . . . . : 192.168.X.X`

2. На смартфоне откройте браузер и введите:
   ```
   http://192.168.X.X:8000
   ```
